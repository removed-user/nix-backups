### 1. Engine-Level Magic Attributes (C++ Interpreter)

These attributes are compiled directly into the C++ binary engine of the Nix language evaluator. The interpreter explicitly looks for these exact string literals on attribute sets to trigger compiler-level shortcuts or alter evaluation behavior.


| Attribute | Category | What the Nix Interpreter Engine Does With It |
| :--- | :--- | :--- |
| `__functor` | Functionality | Converts an attribute set into an executable function call. |
| `__toString` | Coercion | Overrides how the set behaves when evaluated directly inside an interpolation string (e.g., `"${mySet}"`). |
| `outPath` | Coercion | The default fallback path used if `__toString` is not defined; implicitly created by all `derivation` built-ins. |
| `type` | Identification | If `type = "derivation";`, the Nix REPL and build commands change how they print, register, and handle the data layer. |
| `__functionArgs` | Reflection | Overrides the behavior of the `builtins.functionArgs` reflection check, heavily relied on by `pkgs.callPackage`. |
| `__structuredAttrs` | Build Engine | Dictates whether Nix should pass attributes as plain environment string variables or export them cleanly into an unconstrained `.attrs.json` file. |
| `__ignoreNulls` | Build Engine | Instructs the builder engine to silently filter out and omit attributes whose values evaluate to `null` instead of crashing or generating an error. |
| `__id` | Identity | Reserved and internally leveraged by the evaluation compiler engine for tracing identity optimizations. |

### 2. Nixpkgs Framework Attributes (Library Level)

These attributes are not hardcoded into the C++ interpreter engine. Instead, they are conventional keywords checked by the functions and tools inside the `nixpkgs` library or the wider ecosystem (like Hydra and Nix Flakes).


| Attribute | Category | What the Nixpkgs / Ecosystem Code Does With It |
| :--- | :--- | :--- |
| `recurseForDerivations` | Monorepo/CI | Instructs `nix-env`, `nix search`, and Hydra CI to step *inside* the attribute set recursively to find deeper packages. |
| `meta` | Metadata | A standard attribute containing a set of package properties (e.g., `description`, `license`, `maintainers`) used by Nixpkgs tooling and search engines. |
| `passthru` | Package API | An attribute set used inside `stdenv.mkDerivation` to attach fields, scripts, or tests to a package package that shouldn't affect the build derivation hash. |
| `override` / `overrideAttrs` | Customization | Functions attached to packages generated by Nixpkgs to let users easily modify inputs or build steps later. |
| `outputs` | Flakes | The mandatory entry point attribute inside a `flake.nix` file that defines your entire system configurations, packages, and development shells. |
